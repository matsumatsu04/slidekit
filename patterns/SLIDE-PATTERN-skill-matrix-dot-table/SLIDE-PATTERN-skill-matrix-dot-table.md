## Overview

**パターン名：** skill-matrix-dot-table
**概要：** 項目（行）とカテゴリ（列）の対応関係を、記号（●/○/－）だけで一覧できるマトリクス表。文章での説明を省き、対応の有無・度合いを視覚記号で即座に把握させるレイアウト
**適したシーン：** サービス・プラン・機能の対応比較、スキルセットの保有状況一覧、対応可否チェックリストの提示

## Structure（構造）

コンテンツエリアにテーブルを配置する。列構成は「項目名（左端1列）」＋「カテゴリ（4〜5列）」。行数は項目4行が基本。ヘッダー行はカテゴリ名を並べ、面塗り（soft）で他の行と区別する。各データセルは文字列を持たず、対応度合いを表す記号1文字のみを中央揃えで置く。

    structure:
      layout: data-table (matrix)
      columns:
        - name: row-label (item name, left-aligned)
        - name: category-1..category-n (dot cell, centered)
      header-row:
        fill: soft
        content: category names only
      rows: 4 (item names)
      cell_values:
        - filled-dot (該当)
        - outline-dot (一部該当)
        - dash (非該当)

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| カテゴリ見出し | 最上段（ヘッダー行・soft塗り） | 列が何のカテゴリかを示す |
| 項目名 | 各行左端 | 比較対象の項目・機能・サービス名 |
| 記号セル | 各行×各列の交点 | 該当（●）／一部該当（○）／非該当（－）を記号だけで表現 |
| 行区切り線 | 各行間 | 行の境界を明確に示す |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-skill-matrix-dot-tableのレイアウトで、[4つの項目]と[5つのカテゴリ]の対応マトリクスを作成してください。各セルは●（該当）／○（一部該当）／－（非該当）の記号のみで表現してください。デザインはSLIDE.mdに従ってください。」

**注意点：**
- セルの中に説明文字を入れない。記号1文字のみで完結させる（可読性と再現性のため）。
- 行は4行、列は4〜5列までが可読性を保てる上限。それ以上は表を分割するか項目を統合する。
- 記号の意味（●＝該当／○＝一部該当／－＝非該当）は表の外（本文や凡例）で一度説明しておくと誤読を防げる。
- ●と○と－の使用頻度が偏りすぎると一覧性が下がるため、実データの傾向をそのまま反映させる（見た目のために操作しない）。
