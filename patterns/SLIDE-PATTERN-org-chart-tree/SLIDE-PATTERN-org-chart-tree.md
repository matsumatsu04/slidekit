# SLIDE-PATTERN-org-chart-tree

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。SLIDE.mdと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はSLIDE.mdの `Slide Frame` セクションで定義されるため、このファイルには含みません。

## Overview

**パターン名：** org-chart-tree
**概要：** 最上部に1個のトップボックス、その下に第2階層3個、さらに各第2階層の下に第3階層2個ずつ（計6個）を配置する、上から下への3階層組織図。接続線はすべて縦横の直角線のみで構成する。
**適したシーン：** 組織図・体制図の説明、役割分担の階層表示、プロジェクト体制の提示

## Structure（構造）

コンテンツエリア中央に、上から下へ3階層のツリーを配置する。最上段にトップノード1個、中央にその直下の部門ノード3個、下段に各部門の下にチームノード2個ずつを置く。ノード間はすべて垂直線・水平線の直角接続のみで結ぶ（斜め線・曲線は使わない）。

    structure:
      layout: top-down-org-tree
      level-1:
        nodes: 1 (top box, emphasized)
      level-2:
        nodes: 3 (evenly spaced under level-1)
      level-3:
        nodes: 6 (2 per level-2 node)
      connector:
        type: right-angle lines only (vertical stem + horizontal bus)
      constraint: total nodes <= 10 (this pattern uses exactly 1 + 3 + 6 = 10)

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| トップノード | 最上段中央 | 組織全体・プロジェクト全体を代表する最上位の箱 |
| 部門ノード | 第2階層・3個 | トップの直下に位置する主要部門・チーム |
| チームノード | 第3階層・各部門2個 | 部門の下に位置する小チーム・担当領域 |
| 接続線 | 各階層間 | 縦横の直角線のみで階層関係を明示する |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-org-chart-treeのレイアウトで、トップに[組織名]、第2階層に[部門A/B/C]、各部門の下に[チーム名2個ずつ]を配置してください。デザインはSLIDE.mdに従ってください。」

**注意点：**
- **ノード数は最大10個**（トップ1＋第2階層3＋第3階層6）。これを超える場合は第3階層を省略するか、部門数を減らす。
- **接続線は縦・横の直角線のみ**。斜め線・曲線・矢印は使わない。
- 第2階層は必ず3個、第3階層は各部門2個ずつの構成を基本とする。数を変える場合も左右対称になるよう揃える。
- ノードのテキストは短く（部門名・チーム名のみ）。役職者名や詳細説明は入れない。
