## Overview

**パターン名：** column-stacked-tag-list
**概要：** 3〜4列のカラムヘッダー（帯）を横に並べ、各列の下に小さな角丸タグボックスを縦に積み重ねるレイアウト。列ごとに項目数が不揃いでもよい。
**適したシーン：** カテゴリ別の項目一覧、部門別のタスク・担当一覧、分類ごとの用語整理、列数に対して項目数が偏るリスト

## Structure（構造）

コンテンツエリアを3〜4列に等分する。各列の最上部にカラムヘッダー（帯状の見出し）を置き、その下にタグボックスを縦積みする。列間の項目数はそろえなくてよい（4/3/5/2個のように不揃いを許容する）。

    structure:
      layout: column-stacked-tag-list
      columns: 3-4
      column:
        header: band (background fill, centered text)
        tag-stack:
          type: vertical stack
          tag: rounded-box, short-text
          count: variable per column (example: 4 / 3 / 5 / 2)

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| カラムヘッダー | 各列の最上部 | 列が表すカテゴリ名を帯状に表示 |
| タグボックス | ヘッダー下・縦積み | カテゴリに属する個別項目を1つずつ表示 |
| 列 | 横に3〜4本 | カテゴリ単位で情報を独立して整理する |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-column-stacked-tag-listのレイアウトで、[列数3〜4]、各列見出しに[カテゴリ名]、各列の項目に[項目名リスト]を入れてください。列ごとの項目数は揃えなくて構いません。デザインはSLIDE.mdに従ってください。」

**注意点：**
- 列数は3〜4列を推奨。5列以上になると1列の幅が狭くなりタグの文字が折り返しすぎる。
- 1列あたりの項目数は2〜6個を目安にする。極端に多い列がある場合は上位カテゴリで束ねる。
- タグのテキストは短く（10字以内目安）。長文は入れない。
- 列ごとに項目数が違うのは想定どおりの見た目。無理に空欄で埋めて揃える必要はない。
