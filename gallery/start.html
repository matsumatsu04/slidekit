SlideKit 導入手順

SlideKitは、スライドの「デザイン」と「構図」を型として固定し、Claude Codeとの会話だけで 統一感のあるスライド（HTML＋PDF）を生成する仕組みです。導入は3ステップ、5分程度で終わります。

準備するもの

  • Claude Code（ターミナルで claude が起動できる状態）
  • Node.js（バージョン14以上を推奨。無くてもスライド生成は可能ですが、あるとビルドが速く安定します）
  • Google Chrome（PDF書き出しに使用。ブラウザ印刷でも代替できます）

導入（3ステップ）

1

リポジトリを取得する

ターミナルで次を実行します（フォルダはどこでもOK）。

git clone https://github.com/matsumatsu04/slidekit.git

gitを使ったことがない場合は、GitHubのページ右上の「Code → Download ZIP」からダウンロードして解凍しても同じです。

2

スキルを設置する

3つのスキルをClaude Codeのスキル置き場にコピーします。

cp -r slidekit/skills/slidekit-* ~/.claude/skills/

これで、どのフォルダでClaude Codeを起動しても「スライドを作って」が使えるようになります。

3

最初のスライドを作る

取得した slidekit フォルダでClaude Codeを起動し、こう話しかけます。

スライドを作って

タイトル・対象・目的・枚数などのヒアリングに答え、構成案を承認すると、 index.html（そのまま画面表示できるスライド）と deck.pdf が生成されます。

できたスライドの確認・修正・PDF保存

  1. 生成された index.html を スライド確認・修正依頼 のページにドラッグ＆ドロップ
  2. スライドが一覧表示されるので、直したいスライドの下の欄に修正指示を書く（構図を変えたい場合はギャラリーで探したパターンID（P045等）を記入）
  3. 「修正プロンプトを出力」を押してコピーし、そのままClaude Codeに貼り付けると修正版が生成される
  4. 仕上がったら「PDFとして保存（印刷）」からPDF化（印刷ダイアログで「PDFに保存」を選択）
カラーパレット（配色セット）はギャラリー・確認ページのどちらでも試着できます。 気に入った配色があれば、修正プロンプトに自動で含まれるので、そのままClaude Codeに渡せば反映されます。

うまくいかないとき

  • 「スライドを作って」に反応しない → ステップ2のコピー先が ~/.claude/skills/ になっているか確認し、Claude Codeを再起動
  • ビルドでNode.jsのエラーが出る → node -v が14以上か確認。古い場合は最新のLTS版を導入（Claude Codeに「Nodeを更新して」と頼んでもOK）
  • PDFがうまく書き出せない → index.html をChromeで開いて ⌘P →「PDFに保存」でも同じものが得られます
